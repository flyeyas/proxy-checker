# 存储层重构方案 v2

> 状态：✅ 已完成（Phase 1–6 全部落地，测试 140 全通过）  
> 适用版本：proxy-forge v0.1.1+  
> 目标：解决数据散落、运行历史全量重写、checked 集合 O(n²) 增长、命名混乱等问题

---

## 1. 背景与问题

当前存储分散在 4 个按"数据类型"切分的目录：

| 目录 | 文件 | 内容 |
|---|---|---|
| `repo_data/` | `<token>.json` + `<token>.txt` | 仓库（对外 API 载体） |
| `checked_data/` | `<token>.txt` | 已检测代理集合 |
| `auto_data/` | `<token>.json` | auto 任务配置 + 状态 |
| `logs/` | `<token>.json` + `server.log` | 运行历史 + 进程日志混在一起 |

主要问题：

1. **数据按类型分目录**：单个用户的状态散落在 4 处，删除/备份/迁移单个租户都很麻烦
2. **运行历史 JSON 全量重写**：[log_store.py:60](proxy_forge/storage/log_store.py:60) 每写一条都要 read → sort → trim → rewrite，O(n) 写、并发会丢条目
3. **checked 集合无限增长**：[checked_store.py:36](proxy_forge/storage/checked_store.py:36) 全量加载 set，每次 append 都全量重写，长期跑 O(n²) 且无 TTL
4. **auto 收尾跨 4 文件无事务**：中间崩溃会留下不一致状态
5. **命名混乱**：`logs/` 同时存进程日志和产品运行历史
6. **路径默认值坑**：自托管默认 `server.log` 落在项目根目录（已通过修改 `config.json` 默认值修复）

---

## 2. 设计原则

按"性质对号入座"，不一刀切：

| 资源 | 性质 | 存储形式 |
|---|---|---|
| 进程日志 | 文本行流、需轮转、需 grep/tail | `RotatingFileHandler` |
| 运行历史 | 结构化事件、单租户上限千条、追加为主 | **JSONL append-only** |
| 已检测集合 | set 成员关系、需 O(log n) 查询、需 TTL | **SQLite**（标准库内置） |
| auto 配置/状态 | 小、低频、读多写少 | JSON 原子写 |
| 对外仓库 | API 载体、外部消费 | JSON + TXT 原子写 |

**核心约束**：

- `/api/repo/<token>.txt` 和 `/api/repo/<token>.json` URL 不变、字节级兼容（外部程序在订阅）
- 自托管单进程能用、零外部服务依赖（不引入 Postgres/Redis）
- Serverless 路径继续可用
- 每一步可独立 ship、可独立回滚

## 5. 各 backend 实现要点

### 5.1 `RunsBackend`（JSONL append-only）

**写入**：

```python
def insert(self, entry):
    entry = self._compact(entry)
    line = json.dumps(entry, ensure_ascii=False, separators=(",", ":")) + "\n"
    with self._lock:
        with open(self.path, "a", encoding="utf-8") as f:
            f.write(line)
        self._maybe_compact()
    return entry["id"]

def update(self, run_id, fields):
    # 不去找原条目，append 一条 patch
    patch = {"id": run_id, **fields, "_patch": True}
    self.insert(patch)
```

**读取**（按 id 折叠取最新）：

```python
def list(self, limit=None):
    if not os.path.isfile(self.path):
        return []
    merged = {}   # id → 最新合并值
    order = []
    with open(self.path, "r", encoding="utf-8") as f:
        for line in f:
            try:
                item = json.loads(line)
            except json.JSONDecodeError:
                continue                 # 容错损坏行
            rid = item.get("id")
            if not rid:
                continue
            if rid not in merged:
                order.append(rid)
            merged[rid] = {**merged.get(rid, {}), **item}
    items = [self._strip_patch_marker(merged[rid]) for rid in order]
    items.sort(key=lambda i: int(i.get("started_at") or 0), reverse=True)
    return items[:limit] if limit else items
```

**截断**（防无限增长）：

```python
def _maybe_compact(self):
    if self._writes_since_compact < 100:
        self._writes_since_compact += 1
        return
    items = self.list(limit=LOG_LIMIT)
    tmp = self.path + ".tmp"
    with open(tmp, "w", encoding="utf-8") as f:
        for item in reversed(items):
            f.write(json.dumps(item, ensure_ascii=False) + "\n")
    os.replace(tmp, self.path)
    self._writes_since_compact = 0
```

**并发**：单进程内 `threading.Lock`。多进程超出当前架构。

**优势**：
- 是文件、能 `tail -f`、能 `jq` 处理、能 `cp` 备份
- POSIX 单次 `write` < 4096 字节天然原子，不会写半截行
- 摊销 O(1) 写入，每 100 次写做一次 O(n) compact
- 损坏单行不影响其它

**实现**：

```python
class CheckedBackend:
    def __init__(self, tenant_dir):
        self.path = os.path.join(tenant_dir, "checked.db")
        self._local = threading.local()

    def _conn(self):
        if not hasattr(self._local, "conn"):
            conn = sqlite3.connect(self.path, isolation_level=None, check_same_thread=False)
            conn.execute("PRAGMA journal_mode=WAL")
            conn.execute("PRAGMA synchronous=NORMAL")
            conn.execute("PRAGMA busy_timeout=5000")
            conn.executescript(SCHEMA_SQL)
            self._local.conn = conn
        return self._local.conn

    def filter_unchecked(self, proxies):
        if not proxies:
            return []
        keys = [proxy_key(p) for p in proxies]
        conn = self._conn()
        placeholders = ",".join("?" * len(keys))
        rows = conn.execute(
            f"SELECT proxy_key FROM checked_proxies WHERE proxy_key IN ({placeholders})",
            keys,
        ).fetchall()
        checked_set = {row[0] for row in rows}
        return [p for p, k in zip(proxies, keys) if k not in checked_set]

    def add(self, proxies):
        rows = [
            (proxy_key(p), p, int(time.time()))
            for p in normalize_proxy_list(proxies)
        ]
        if not rows:
            return
        self._conn().executemany(
            "INSERT OR REPLACE INTO checked_proxies(proxy_key, proxy, checked_at) VALUES (?,?,?)",
            rows,
        )

    def prune(self, retention_days, max_rows):
        cutoff = int(time.time()) - retention_days * 86400
        conn = self._conn()
        conn.execute("DELETE FROM checked_proxies WHERE checked_at < ?", (cutoff,))
        conn.execute("""
            DELETE FROM checked_proxies WHERE rowid NOT IN (
                SELECT rowid FROM checked_proxies ORDER BY checked_at DESC LIMIT ?
            )
        """, (max_rows,))
```

```python
from logging.handlers import RotatingFileHandler

def configure_logging(log_file_path, logger_name="proxy_forge"):
    logger = logging.getLogger(logger_name)
    if getattr(logger, "_proxy_forge_configured", False):
        return logger
    logger.setLevel(logging.INFO)
    formatter = logging.Formatter("%(asctime)s [%(levelname)s] %(message)s")

    stream_handler = logging.StreamHandler()
    stream_handler.setFormatter(formatter)

    directory = os.path.dirname(os.path.abspath(log_file_path))
    if directory:
        os.makedirs(directory, exist_ok=True)
    file_handler = RotatingFileHandler(
        log_file_path,
        maxBytes=10 * 1024 * 1024,
        backupCount=5,
        encoding="utf-8",
    )
    file_handler.setFormatter(formatter)

    logger.addHandler(stream_handler)
    logger.addHandler(file_handler)
    logger.propagate = False
    logger._proxy_forge_configured = True
    return logger
```

## 7. 迁移脚本

新增 `proxy_forge/migrate/v1_to_v2.py`：

```python
def migrate_tenant(token, base_dir):
    new_dir = os.path.join(base_dir, "data", token)
    if os.path.isdir(new_dir) and os.listdir(new_dir):
        return "skipped"           # 幂等

    os.makedirs(new_dir, exist_ok=True)

    # repo
    legacy_repo_json = os.path.join(base_dir, "repo_data", f"{token}.json")
    legacy_repo_txt = os.path.join(base_dir, "repo_data", f"{token}.txt")
    if os.path.isfile(legacy_repo_json):
        shutil.copy2(legacy_repo_json, os.path.join(new_dir, "repo.json"))
    if os.path.isfile(legacy_repo_txt):
        shutil.copy2(legacy_repo_txt, os.path.join(new_dir, "repo.txt"))

    # auto
    legacy_auto = os.path.join(base_dir, "auto_data", f"{token}.json")
    if os.path.isfile(legacy_auto):
        data = read_json_file(legacy_auto, {})
        if isinstance(data, dict):
            data.get("state", {}).pop("history", None)
        atomic_write_json(os.path.join(new_dir, "auto.json"), data)

    # runs: logs/<token>.json (数组) → runs.jsonl
    legacy_runs = os.path.join(base_dir, "logs", f"{token}.json")
    if os.path.isfile(legacy_runs):
        items = read_json_file(legacy_runs, [])
        if isinstance(items, list):
            with open(os.path.join(new_dir, "runs.jsonl"), "w") as f:
                for item in items:
                    f.write(json.dumps(item, ensure_ascii=False) + "\n")

    # checked: checked_data/<token>.txt → checked.db
    legacy_checked = os.path.join(base_dir, "checked_data", f"{token}.txt")
    if os.path.isfile(legacy_checked):
        backend = CheckedBackend(new_dir)
        with open(legacy_checked, "r", encoding="utf-8") as f:
            proxies = [line.strip() for line in f if line.strip()]
        backend.add(proxies)
        backend.close()

    return "migrated"


def list_legacy_tokens(base_dir):
    tokens = set()
    for sub, ext in [("repo_data", ".json"), ("checked_data", ".txt"),
                     ("auto_data", ".json"), ("logs", ".json")]:
        d = os.path.join(base_dir, sub)
        if not os.path.isdir(d):
            continue
        for name in os.listdir(d):
            base, e = os.path.splitext(name)
            if e == ext and base:
                tokens.add(sanitize_token(base))
    return sorted(tokens)


def run(base_dir, *, dry_run=False):
    tokens = list_legacy_tokens(base_dir)
    results = {}
    for token in tokens:
        if dry_run:
            results[token] = "would_migrate"
        else:
            results[token] = migrate_tenant(token, base_dir)
    return results
```

## 10. 一致性策略

auto 一轮收尾的写顺序：

```python
# auto_coordinator_service.execute_run 末尾
storage.checked.add(detected)          # 1. 已检测记录
storage.runs.update(run_id, summary)   # 2. 运行摘要
storage.auto.write(updated)            # 3. 配置+状态
storage.repo.write(merged)             # 4. 仓库（外部消费）
```

无事务，但**每一步幂等且可重放**：

- step 1 用 `INSERT OR REPLACE`，重跑只刷 `checked_at`
- step 2 在 JSONL 里 append 一条 patch，按 id 合并最新生效
- step 3 是覆盖式 atomic write
- step 4 用 `merge_repo_data` 合并，重跑结果一致

崩溃恢复（启动时跑一次）：

- `runs.jsonl` 里有 `status="running"` 但 `auto.state.running=false` → 标 `interrupted`
- `auto.state.running=true` 但当前进程无 thread → reset，下次 schedule 重跑

这部分代码已经在 `mark_interrupted_runs` 雏形里，把它对接到新存储即可。

## 13. 落地建议

1. **先做 Phase 1+2** — 这两步零行为变化，只是收敛接口。所有存储调用统一通过 `TenantStorage`，未来任何替换都只改一处。
2. **观察一周** — 跑现有测试 + 手工 smoke，确认接口抽象没漏。
3. **再做 Phase 3** — 目录迁移是用户感知最强的一步，单独发布、单独写升级文档。
4. **Phase 4/5/6 顺序无依赖**，按重要性可调整：
   - 优先 4（runs.jsonl）：收益是消除日志全量重写，立刻见效
   - 次 5（checked.db）：收益是消除 O(n²)，长期重要
   - 6 最后做清理

## 15. 风险登记

| 风险 | 等级 | 缓解 |
|---|:---:|---|
| Docker volume 用户升级时遗漏改 `docker-compose.yml` | 中 | README 升级章节 + 启动检测旧目录有数据时输出明显警告 |
| 自动迁移在大量租户时启动变慢 | 低 | 单租户迁移耗时 < 100ms，1000 租户也只是 1~2 分钟，且只跑一次 |
| SQLite 在某些 Docker 镜像下 WAL 模式失效（OverlayFS） | 低 | 检测到 WAL 失败时降级为 `journal_mode=DELETE` |
| JSONL compact 时崩溃 | 低 | 写 tmp + `os.replace` 原子，最坏丢一条本次 append |
| Serverless（Vercel）只读 FS 路径 | 低 | `serverless.py` 不依赖 `data/`，保持现状 |
