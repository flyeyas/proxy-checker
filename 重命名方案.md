# 项目重命名方案:Proxy Checker → ProxyForge

## 背景

`proxy-checker` 名字只覆盖"检测"单一功能,但项目自 v6.0 起已是「检测 + 筛选 + 仓库维护 + 后台自动任务 + 统一转发网关」的完整面板;且 GitHub 上同名项目过多、辨识度低、易与上游 fork 源混淆。

新名定为 **ProxyForge**,突出代理池/仓库定位,做全量重命名(含 Python 包名)。本机无任何运行数据,因此运行时标识符(cookie / localStorage / docker 卷)可一并重置。

## 命名映射(4 种形态)

| 当前 | 改为 | 用途 |
|---|---|---|
| `proxy_checker` | `proxy_forge` | Python 包名、import、cookie 名、docker 卷名、localStorage 键、日志属性 |
| `proxy-checker` | `proxy-forge` | docker 镜像名、container_name、GitHub 仓库地址 |
| `Proxy Checker` | `ProxyForge` | README/PRODUCT 展示名、日志启动文案 |

**利好:** 带后缀的标识符(`proxy_checker_auth`、`proxy_checker_repo` 卷名、`proxy_checker_token` localStorage 等)都以 `proxy_checker` 开头,统一替换 `proxy_checker → proxy_forge` 一次即可全部覆盖。

## 执行步骤

### 1. 重命名包目录(`git mv` 保留历史)
```
proxy_checker/  →  proxy_forge/
```

### 2. 批量文本替换(按精确顺序,避免嵌套误伤)
- 全仓 `proxy_checker` → `proxy_forge`(94 个文件,266 处 import + cookie + 卷名 + localStorage 键 + `_proxy_checker_configured` 日志属性)
- 全仓 `proxy-checker` → `proxy-forge`(docker-compose ×2、CI、.env.example、README、app.js、index.html)
- 展示名 `Proxy Checker` → `ProxyForge`(README、PRODUCT.md、`server_runner.py` 启动日志、各 `__init__.py` docstring、`api/index.py`)

### 3. 不改动的部分(与项目名无关)
- 运行期目录 `data/`、`logs/`(`DATA_DIR` 不含项目名)
- `LEGACY_REPO_DIR` 等 `repo_data/checked_data/auto_data` 常量(迁移工具用的旧路径)
- 根目录 shim `proxy_check.py`、`fetch_proxies.py`(文件名不含 `proxy_checker`)
- 环境变量 `PROXY_GATEWAY_*`、`PROXY_HOST_*`(网关配置,非项目名)
- 版本号 `v0.1.1`

### 4. 验证
- `conda activate xiangmu && pytest`(当前 140 passed 须保持全绿)
- `grep -ri "proxy_checker\|proxy-checker\|Proxy Checker"` 确认零残留
- `python server.py` 冒烟确认能起服务

### 5. 提交(当前 `refactor/storage-layer-v2` 分支)
```
refactor: rename project proxy-checker → proxy-forge (ProxyForge)
```
